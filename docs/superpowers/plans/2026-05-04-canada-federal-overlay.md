# Canada Federal Overlay Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the 12-command **Canada Federal Overlay** as ArcKit v4.14.0 community overlay — community count goes from 34 → 46, total from 104 → 116. Flat overlay (no plugin `userConfig` changes, no agents, no hooks). Each command produces a standard ArcKit artefact under the existing Document Control pattern.

**Architecture:** 12 sibling community-tagged commands prefixed `ca-*`, each with a paired template and guide. Generated extension formats (Codex / OpenCode / Gemini / Copilot) come out of `scripts/converter.py` unchanged. The Canadian classification scheme (`UNCLASSIFIED / Protected A / Protected B / Protected C / CONFIDENTIAL / SECRET / TOP SECRET`) is rendered per-template, not via plugin userConfig — keeping the overlay drop-in.

**Tech Stack:** Markdown command/template/guide authoring, JavaScript (`doc-types.mjs`), Python (`scripts/converter.py`), `markdownlint-cli2`, bash helpers under `scripts/bash/`.

**Design Spec:** `docs/superpowers/specs/2026-05-04-canada-federal-overlay-design.md`

**Branch:** Already on `feat/ca-overlay-spec` (cut from `origin/main`). All commits land here. **Do not push** — the work is local-only until the user explicitly authorises a push.

**Sensitivity:** Spec and plan are local-only. No GitHub Issues, no PRs, no `/arckit:*` MCP calls that surface project context externally, no test-repo updates. If a step would push to a remote or open an issue, **stop and check with the user first**.

# PHASE B — COMMANDS, TEMPLATES, GUIDES

Goal: ship the 12 `ca-*` commands. Each task creates one command file, one template (twice — `arckit-claude/templates/` and `.arckit/templates/` — they are byte-identical mirrors), and one guide (twice — `docs/guides/` and `arckit-claude/docs/guides/`).

**Common steps for every Phase B task** (referenced as "shared B steps"):

1. **Read peer commands** for shape: `arckit-claude/commands/uae-pdpl.md` is the closest community-overlay reference. Match its structure (process steps, RENDERING.md reference, write-tool output, summary-only-to-user).
2. **Frontmatter** — uses the shape in "Reusable content fragments" above. Pull `description`, `handoffs`, and `keep-coding-instructions` from spec §3.
3. **Statutory currency banner** — paste the appropriate fragment (FITAA-specific banner for FITAA/SOIA/AIA/Charter/OCAP; standard community banner otherwise).
4. **Process** — every command follows this skeleton:
    ```markdown
    ## Process

    1. Read prerequisites:
       - `projects/000-global/ARC-000-PRIN-*.md` (architecture principles, if present)
       - <command-specific upstream artefacts — see per-command list below>
       - `${CLAUDE_PLUGIN_ROOT}/templates/_partials/RENDERING.md`
    2. Read the template:
       - **First**, check `.arckit/templates-custom/<name>-template.md` (user override)
       - **Then**, `.arckit/templates/<name>-template.md`
       - **Fallback**, `${CLAUDE_PLUGIN_ROOT}/templates/<name>-template.md`
    3. Use `scripts/bash/generate-document-id.sh <TYPE-CODE> --filename` for the artefact filename.
    4. Resolve the `<!-- DOC-CONTROL-HEADER -->` marker per `RENDERING.md`.
    5. Generate content per the section structure listed below.
    6. Use the **Write** tool to save under `projects/<project-dir>/`.
    7. Show only a summary to the user (3–5 bullets).
    ```
5. **Verification**: run `markdownlint-cli2 arckit-claude/commands/<name>.md arckit-claude/templates/<name>-template.md .arckit/templates/<name>-template.md docs/guides/<name>.md arckit-claude/docs/guides/<name>.md` after each task.
6. **Commit** at the end of each task with message `feat(ca-overlay): add /arckit:<name>`.
